# Swiss Kit 产品路线图

> 精品开发者工具集 — 中期规划（3-6 个月）

## 产品定位

Swiss Kit 是一个精品本地开发者工具集，追求"少即是多"。每个工具都打磨到位，聚焦一个明确任务，能在 3 秒内完成最常见操作。

**当前状态**：4 个核心工具，完善的主题系统（5 套主题），可扩展的工具注册架构。

**目标状态**：14 个精品工具，覆盖开发者日常高频场景。

## 核心原则

- 每个工具必须能在 3 秒内完成最常见操作
- 不做功能堆砌，每个工具聚焦一个明确的任务
- 新工具上线前必须有完整的单元测试和 5 套主题适配
- YAGNI：不为假设性需求增加复杂度

## 里程碑总览

| 里程碑 | 主题         | 工具数量 | 重点                              |
| ------ | ------------ | -------- | --------------------------------- |
| **M1** | 格式转换核心 | 4 → 8    | 高频格式转换 + 时间/进制/色值工具 |
| **M2** | 实用工具补全 | 8 → 12   | 更广场景覆盖 + 分类标签体系       |
| **M3** | 精品化收尾   | 12 → 14  | 收尾工具 + 交互打磨 + 分发优化    |

## M1：格式转换核心（4 → 8 个工具）

聚焦**数据格式互转**和**开发者常用数值工具**，与现有 JSON 格式化工具形成自然的工具族群。

### 新增工具

#### 1. JSON ↔ YAML 转换器

- 左右分栏布局：左侧 JSON，右侧 YAML，双向实时转换
- 支持语法高亮和错误提示
- 支持缩进风格设置（2/4 空格）
- **优先理由**：YAML 在 K8s/Docker/CI 配置中无处不在，和 JSON 工具天然互补

#### 2. 时间戳转换器

- Unix 时间戳（秒/毫秒）↔ 可读日期时间
- 显示"距今 X 天/小时"的相对时间
- 支持多种日期格式（ISO 8601、RFC 2822、自定义格式）
- 内置"当前时间"快捷按钮
- **优先理由**：调试日志、API 响应中时间戳转换是刚需

#### 3. 进制转换器

- 二进制 / 八进制 / 十进制 / 十六进制 互转
- 支持批量转换（多行输入）
- 同步显示所有进制的结果
- **优先理由**：操作简单、实现成本低，能快速充实工具数量

#### 4. 颜色转换器/选择器

- HEX ↔ RGB ↔ HSL ↔ CSS Named Colors 互转
- 可视化颜色预览面板
- 支持调色板和渐变预览
- 一键复制各格式的颜色值
- **优先理由**：前端开发者日常高频使用，且视觉效果好，提升产品调性

### M1 附带改进

- 首页工具磁贴在 8 个工具下的 Grid 布局优化
- 每个新工具匹配 5 套主题的渐变配色（`--tool-xxx-gradient` CSS 变量）

## M2：实用工具补全（8 → 12 个工具）

在格式转换基础上，补充**更广泛的实用场景**，让工具集从"格式处理"扩展到"日常开发全场景"。

### 新增工具

#### 5. JSON ↔ XML 转换器

- 双向转换，保留属性和嵌套结构
- 支持格式化输出（缩进控制）
- 与 M1 的 JSON ↔ YAML 形成"格式转换三件套"
- **理由**：使用频次略低于 YAML，但对全栈/后端开发者仍有价值

#### 6. CSV/TSV 查看器与转换器

- CSV/TSV 文本 → 表格可视化展示
- 支持 CSV ↔ JSON 互转
- 支持分隔符自定义（逗号、Tab、分号）
- 支持拖拽文件上传（首个支持文件输入的工具）
- **理由**：引入"文件上传"交互模式，为后续工具打下基础

#### 7. Markdown ↔ HTML 转换器

- Markdown 实时渲染为 HTML 预览
- HTML → Markdown 反向转换
- 支持 GFM（GitHub Flavored Markdown）
- **理由**：与前端开发者写文档、博客的场景紧密结合

#### 8. 数字/单位转换器

- 存储单位：KB ↔ MB ↔ GB ↔ TB
- 数字格式：千分位、科学计数法、中文大写
- CSS 单位：px ↔ rem ↔ em（可设置 base font-size）
- **理由**：轻量实用，配合进制转换器形成"数字工具组"

### M2 附带改进

- **工具分类标签体系**：格式转换 / 编解码 / 数字时间 / 文本处理
- 首页支持按分类筛选工具
- 首个文件拖拽上传基础组件（由 CSV 工具驱动）

## M3：精品化收尾（12 → 14 + 体验打磨）

M3 不再大量扩展，而是**收尾补缺 + 整体打磨**，让 Swiss Kit 达到"精品"标准。

### 新增工具

#### 9. Cron 表达式解析器

- 输入 Cron 表达式，显示可读描述（如"每周一上午 9 点"）
- 展示接下来 5-10 次执行时间
- 支持反向生成：选择时间规则 → 生成 Cron 表达式
- **理由**：DevOps/后端刚需，实现精巧，提升工具集专业度

#### 10. 文本 Diff 对比器

- 左右分栏输入两段文本
- 逐行/逐字差异高亮（增/删/改用不同颜色标注）
- 支持忽略空白、大小写选项
- **理由**：使用频次高、技术挑战适中，视觉呈现效果好

### M3 体验打磨

| 改进项             | 内容                                                    |
| ------------------ | ------------------------------------------------------- |
| **全局键盘快捷键** | `Ctrl/Cmd + K` 打开工具搜索，`Ctrl/Cmd + 数字` 切换工具 |
| **历史记录**       | 本地缓存最近 10 次输入（localStorage），支持一键恢复    |
| **响应式适配**     | 平板和窄屏下侧边栏自动折叠，工具页切换为上下布局        |
| **工具搜索**       | 首页搜索框，支持模糊匹配工具名和描述                    |
| **路由级懒加载**   | 确保首屏只加载当前工具，优化加载性能                    |

### M3 分发准备

| 项目            | 内容                                              |
| --------------- | ------------------------------------------------- |
| **npm 包优化**  | 确保 `npx swiss-kit` 首次运行体验流畅，优化包体积 |
| **README 完善** | 工具列表 + GIF 动画展示 + 使用说明                |

## 技术约束

### 每个新工具的交付标准

1. **工具目录结构**：`src/app/tools/<name>/` 包含 `index.ts`、`XxxTool.vue`、`utils.ts`、`__tests__/`
2. **ToolMeta 注册**：在 `registry.ts` 中注册，路由和菜单自动生成
3. **主题适配**：5 套主题的 `--tool-xxx-gradient` CSS 变量
4. **单元测试**：核心逻辑（utils）必须有完整测试覆盖
5. **无障碍**：关键交互元素添加 aria 属性

### 架构扩展点

- M2 引入工具分类时，扩展 `ToolMeta` 接口添加 `category` 字段
- M2 引入文件上传时，创建 `FileDropZone` 共享组件
- M3 引入历史记录时，创建 `useHistory` 组合式 API
- M3 引入快捷键时，创建 `useShortcuts` 组合式 API
